# Отчёт по Спринту 3 — MicroPKI

**Цель спринта:** добавить управление жизненным циклом сертификатов через SQLite, уникальные serial, и HTTP-репозиторий для выдачи сертификатов и CRL endpoint-заглушки.

---

## 1) Что реализовано

### Новые модули
- `micropki/database.py` — схема БД, CRUD-операции, фильтрация и status update stub.
- `micropki/serial.py` — генератор уникальных serial (64-bit composite: timestamp + CSPRNG), проверка уникальности в БД.
- `micropki/repository.py` — HTTP сервер и endpoints:
  - `GET /certificate/<serial>`
  - `GET /ca/root`
  - `GET /ca/intermediate`
  - `GET /crl` -> `501 Not Implemented`

### Изменения в CLI
- `micropki db init --db-path ...`
- `micropki ca list-certs [--status valid|revoked|expired] [--format table|json|csv]`
- `micropki ca show-cert <serial> --db-path ...`
- `micropki repo serve --host --port --db-path --cert-dir`
- `micropki repo status --host --port` (опциональная проверка доступности порта)

### Прозрачная интеграция БД в issuance
- `ca issue-intermediate` и `ca issue-cert` теперь автоматически:
  1. инициализируют БД при необходимости,
  2. генерируют уникальный serial через `serial.generate_unique_serial`,
  3. вставляют запись в таблицу `certificates`,
  4. затем сохраняют файлы на диск.

Это соответствует требованию атомарности “насколько возможно”: если вставка в БД падает — файлы ещё не записаны.

## 2) Схема БД

Создаётся таблица:

```sql
CREATE TABLE IF NOT EXISTS certificates (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    serial_hex TEXT UNIQUE NOT NULL,
    subject TEXT NOT NULL,
    issuer TEXT NOT NULL,
    not_before TEXT NOT NULL,
    not_after TEXT NOT NULL,
    cert_pem TEXT NOT NULL,
    status TEXT NOT NULL,
    revocation_reason TEXT,
    revocation_date TEXT,
    created_at TEXT NOT NULL
);
```

Индексы:
- `idx_certificates_serial`
- `idx_certificates_status`

---

## 3) HTTP репозиторий

Реализовано:
- Логирование каждого HTTP запроса на INFO в формате `[HTTP] METHOD PATH from IP -> STATUS`.
- CORS заголовок: `Access-Control-Allow-Origin: *`.
- Корректные ошибки:
  - `400` для неверного serial формата
  - `404` для отсутствующих сущностей
  - `405` для неподдерживаемых методов
  - `501` для `/crl`
