# Отчет по Спринту 5: OCSP Responder

## Что было реализовано

В рамках 5-го спринта мы добавили возможность проверки сертификатов в режиме реального времени — OCSP (Online Certificate Status Protocol).

### 1. Архитектура файлов и модулей
* Создан выделенный модуль `micropki/ocsp.py`, отвечающий за:
  * Декодирование ASN.1/DER запроса OCSP `v1`.
  * Валидацию запроса (проверка привязки к Issuer).
  * Формирование защищенного от Replay-атак ответа (Echo Nonce).
  * Формирование криптографической подписи ответа `OCSPResponse`.
* Создан сервер-ответчик `micropki/ocsp_responder.py` на базе `FastAPI`. 
  * Он принимает POST и GET запросы для endpoints `/` и `/{base64}`.

### 2. Сертификаты подписи (OCSPSigning)
* Шаблонизатор `micropki/templates.py` теперь понимает шаблон `ocsp`.
* Для него выставлен `ExtendedKeyUsage = id-kp-OCSPSigning`, а `BasicConstraints = CA:FALSE` согласно `RFC 6960` и `OSC-1`.
* Сертификат генерируется командой `micropki ca issue-ocsp-cert`.

### 3. Интеграция с базой данных
При обработке каждого OCSP запроса (по его `serial_number`), сервер читает базу:
1. `valid` -> **Good**
2. `revoked` -> **Revoked** (включая причину и точное время отзыва)
3. Не найдено (или несовпадение) -> **Unknown**

### 4. CLI Команды
Добавлены команды:
* `micropki ca issue-ocsp-cert` — специальная функция, выдающая сертификат OCSP Responder'у без шифрования приватного ключа.
* `micropki ocsp serve` — запуск OCSP провайдера.

## Статус
Все требования выполнены. Код готов к демонстрации ОССП с помощью встроенного в систему `openssl ocsp`.
